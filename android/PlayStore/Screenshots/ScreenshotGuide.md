# ephenotes Android Screenshots Guide

## Overview
This guide provides specifications and guidelines for creating screenshots for the Google Play Store listing.

## Screenshot Requirements

### Play Store Requirements

**Phone Screenshots (Required)**
- **Minimum:** 2 screenshots
- **Maximum:** 8 screenshots
- **Aspect Ratio:** 16:9 or 9:16
- **Minimum Dimension:** 320 pixels
- **Maximum Dimension:** 3840 pixels
- **Format:** JPG or 24-bit PNG (no alpha)
- **Recommended:** 1080 x 1920 pixels (9:16 portrait)

**7-inch Tablet Screenshots (Optional but Recommended)**
- **Minimum:** 2 screenshots
- **Maximum:** 8 screenshots
- **Recommended Size:** 1024 x 600 or 1920 x 1200 pixels
- **Format:** JPG or 24-bit PNG

**10-inch Tablet Screenshots (Optional but Recommended)**
- **Minimum:** 2 screenshots
- **Maximum:** 8 screenshots
- **Recommended Size:** 1280 x 800 or 2560 x 1600 pixels
- **Format:** JPG or 24-bit PNG

## Screenshot Strategy

### Primary Goals
1. **Show key features** clearly and attractively
2. **Tell a story** about the app's value
3. **Highlight unique selling points** (privacy, priority system)
4. **Demonstrate ease of use** with clear UI
5. **Build trust** with professional presentation

### Screenshot Order (Priority)
1. **Main Screen** - Note list with organized notes
2. **Note Creation** - Creating a new note
3. **Priority Organization** - Notes organized by priority
4. **Search** - Search functionality in action
5. **Color Coding** - Notes with different colors
6. **Archive** - Archive screen with restore option
7. **Note Editor** - Editing a note with formatting
8. **Dark Mode** - App in dark mode (optional)

## Screenshot Specifications

### Phone Screenshots

**Recommended Devices for Capture:**
- Google Pixel 7 Pro (1440 x 3120)
- Samsung Galaxy S23 (1080 x 2340)
- OnePlus 11 (1440 x 3216)

**Capture Settings:**
- **Resolution:** Native device resolution
- **Orientation:** Portrait (9:16)
- **Status Bar:** Clean (no notifications)
- **Navigation:** Hide or show consistently
- **Time:** 9:41 AM (iOS convention) or 10:00 AM
- **Battery:** Full or charging
- **Signal:** Full bars, WiFi on

**Resize To:**
- 1080 x 1920 pixels (standard)
- Or 1440 x 2560 pixels (high-res)

### Tablet Screenshots

**Recommended Devices for Capture:**
- Samsung Galaxy Tab S8 (2560 x 1600)
- Google Pixel Tablet (2560 x 1600)

**Capture Settings:**
- **Resolution:** Native device resolution
- **Orientation:** Landscape preferred, portrait optional
- **Layout:** Show tablet-optimized layout
- **Content:** More notes visible on larger screen

## Design Guidelines

### Visual Consistency
- **Font:** Use system font (Roboto)
- **Colors:** Match app's color scheme
- **Spacing:** Consistent padding and margins
- **Alignment:** Proper alignment of elements
- **Shadows:** Subtle elevation shadows

### Content Quality
- **Realistic:** Use realistic, relatable note content
- **Diverse:** Show variety of use cases
- **Professional:** No typos or errors
- **Appropriate:** Family-friendly content
- **Localized:** English for primary market

### Technical Quality
- **Resolution:** High resolution, no pixelation
- **Compression:** Optimize file size without quality loss
- **Format:** JPG for photos, PNG for graphics
- **Color Space:** sRGB
- **Aspect Ratio:** Exact 16:9 or 9:16

## Screenshot Annotations (Optional)

### Annotation Style
- **Font:** Bold, sans-serif (e.g., Roboto Bold)
- **Size:** Large enough to read on mobile
- **Color:** High contrast (white text on dark background or vice versa)
- **Background:** Semi-transparent overlay or solid color
- **Arrows:** Simple, clear arrows pointing to features
- **Position:** Top or bottom, not covering important UI

### Annotation Examples
- "Quick capture in seconds"
- "Organize by priority"
- "Find notes instantly"
- "Your data stays private"
- "Works completely offline"
- "Beautiful Material Design"

### When to Use Annotations
- **Pros:** Clarify features, guide users, highlight benefits
- **Cons:** Can look cluttered, may not translate well
- **Recommendation:** Test both annotated and clean versions

## Capture Methods

### Method 1: Device Screenshots
**Pros:** Authentic, shows real app
**Cons:** Requires real device, status bar cleanup

**Steps:**
1. Install app on device
2. Prepare sample data
3. Clean status bar (time, battery, etc.)
4. Take screenshots (Power + Volume Down)
5. Transfer to computer
6. Edit and resize as needed

### Method 2: Android Emulator
**Pros:** Easy to control, consistent
**Cons:** May not look as authentic

**Steps:**
1. Set up Android emulator in Android Studio
2. Configure device (Pixel 7 Pro recommended)
3. Install app
4. Prepare sample data
5. Take screenshots (camera icon in emulator)
6. Edit and resize as needed

### Method 3: Design Tools (Mockups)
**Pros:** Perfect control, professional look
**Cons:** Time-consuming, requires design skills

**Tools:**
- Figma with device frames
- Sketch with device mockups
- Adobe XD with device previews
- Placeit or similar mockup generators

**Steps:**
1. Export app screens
2. Import into design tool
3. Place in device frame
4. Add annotations if desired
5. Export at required resolution

## Screenshot Editing

### Recommended Tools
- **Figma:** Free, web-based, collaborative
- **Adobe Photoshop:** Professional, powerful
- **GIMP:** Free, open-source alternative
- **Sketch:** Mac-only, popular for UI design
- **Canva:** Easy, template-based

### Editing Checklist
- [ ] Crop to exact aspect ratio
- [ ] Resize to recommended dimensions
- [ ] Clean up status bar
- [ ] Adjust brightness/contrast if needed
- [ ] Add annotations (if using)
- [ ] Optimize file size
- [ ] Convert to JPG or PNG
- [ ] Verify quality at 100% zoom
- [ ] Test on mobile device preview

### File Optimization
- **JPG Quality:** 85-95% (balance quality and size)
- **PNG Compression:** Use tools like TinyPNG
- **Target Size:** < 2 MB per screenshot
- **Color Profile:** sRGB
- **Metadata:** Remove EXIF data

## Upload Checklist

### Before Upload
- [ ] All screenshots meet size requirements
- [ ] File formats are correct (JPG or PNG)
- [ ] File sizes are reasonable (< 2 MB each)
- [ ] Screenshots are in correct order
- [ ] Content is appropriate and professional
- [ ] No sensitive or personal information visible
- [ ] Screenshots represent current app version
- [ ] Quality reviewed by team

### Upload Process
1. Log in to Play Console
2. Navigate to Store listing
3. Scroll to "Phone screenshots" section
4. Upload screenshots in order
5. Verify preview
6. Repeat for tablet screenshots (if applicable)
7. Save changes

- Seasonal or promotional updates

### Update Process

2. Capture new screenshots

4. Review and approve
5. Upload to Play Console
